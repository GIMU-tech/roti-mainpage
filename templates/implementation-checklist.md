# Implementation Checklist

## Before Start
- [ ] Design.md를 읽었다.
- [ ] 이번 단계의 목표를 확인했다.
- [ ] 수정 범위를 명확히 했다.

## During Build
- [ ] 파일 책임이 명확하게 분리되어 있다.
- [ ] 브랜드 데이터가 분리되어 있다.
- [ ] UI와 애니메이션 로직이 과도하게 섞이지 않았다.
- [ ] 카드 클릭 기반 전환 구조가 유지된다.

## Before Finish
- [ ] 헤더가 상단에 안정적으로 배치된다.
- [ ] Hero가 브랜드 포털처럼 보인다.
- [ ] 각 브랜드 풀스크린 섹션이 구분된다.
- [ ] CTA가 명확하다.
- [ ] 레드 포인트 사용이 과하지 않다.
- [ ] 반응형 문제를 기본 점검했다.
- [ ] Design.md 기준 점검을 수행했다.
