# ROTI Mobile Brand Scene Impact Note

작성일: 2026-07-07

## 현재 모바일 브랜드 전환 기준

- 데스크톱 브랜드 장면은 기존 pinned / stacked GSAP 구조를 유지한다.
- 900px 이하 모바일과 태블릿도 데스크톱과 같은 사용감을 목표로 한다.
- 모바일 첫 브랜드 진입용 별도 entry 구간은 제거한다.
- Hero에서 브랜드 섹션으로 내려오면 ROTI CAMP 장면이 한 화면에 바로 걸린다.
- 이후 브랜드 전환은 다음 슬라이드 전체가 아래에서 올라오는 방식으로 처리한다.
- 내부 프레임은 PC처럼 작게 잡힌 카드 형태에서 풀스크린으로 펴진다.
- 전환 중 위쪽 장면은 최대 70%까지 줄고, 아래에서 올라오는 다음 장면은 70%에서 100%로 커진다.
- 같은 브랜드 배경 위에 같은 브랜드 프레임이 중복으로 보이지 않도록, 모바일 슬라이드 배경은 검정 무대로 유지하고 브랜드 이미지는 프레임 안에서만 보여준다.
- ScrollTrigger snap은 각 브랜드 장면 완료 지점에 맞춘다.

## 6단계 확인 항목

- 360px, 390px, 430px, 768px에서 ROTI CAMP / ROTI HOMESYS / LEEL 3개 장면이 모두 보이는지 확인
- Hero에서 ROTI CAMP로 내려갈 때 PC처럼 첫 브랜드 장면이 한 화면에 바로 걸리는지 확인
- 브랜드 전환 때 PC처럼 다음 슬라이드 전체가 아래에서 올라오는지 확인
- 브랜드 전환 중 위쪽 장면이 약 70%까지 줄고, 아래쪽 장면이 70%에서 100%로 커지는지 확인
- 브랜드 전환 후 멈춘 상태에서 이전 브랜드 배경과 다음 브랜드 카드가 겹쳐 보이지 않는지 확인
- 스크롤을 멈췄을 때 다음 브랜드가 살짝 올라와 있는 중간 상태가 아니라 한 브랜드 장면만 완성된 상태로 보이는지 확인
- 로고와 문구가 이미지 위에서 잘리거나 눌리지 않는지 확인
- 터치 스크롤이 과하게 튀거나 멈추지 않는지 확인

## 7단계 영향

- `BrandSlideStack` 안의 desktop pinned timeline 분리
- `BrandSlideStack` 안의 mobile stacked timeline 분리
- 최종적으로 `useBrandSlideStackAnimation` 같은 훅으로 이동
- reduced-motion 처리를 한 위치에서 공통 관리

## 주의

- 모바일 구조는 데스크톱과 같은 인상을 우선한다.
- ROTI CAMP / ROTI HOMESYS / LEEL 3개 브랜드는 항상 모두 보여야 한다.
- 모바일에서 blank scene이나 두 브랜드 겹침이 생기면 P1 blocker로 본다.
